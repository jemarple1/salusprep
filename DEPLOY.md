# SalusPrep — Cloud Deployment

SalusPrep deploys as a standard Laravel 13 application. Recommended platforms: **Laravel Cloud**, **Railway**, or **Render**.

## 1. Stripe setup

1. Create a [Stripe account](https://dashboard.stripe.com/register).
2. In **Developers → API keys**, copy:
   - Publishable key → `STRIPE_KEY`
   - Secret key → `STRIPE_SECRET`
3. In **Developers → Webhooks**, add endpoint:
   ```
   https://YOUR-DOMAIN.com/stripe/webhook
   ```
   Event: `checkout.session.completed`
4. Copy the webhook signing secret → `STRIPE_WEBHOOK_SECRET`

### Production `.env` values

```env
APP_NAME=SalusPrep
APP_ENV=production
APP_DEBUG=false
APP_URL=https://your-domain.com

STRIPE_KEY=pk_live_...
STRIPE_SECRET=sk_live_...
STRIPE_WEBHOOK_SECRET=whsec_...

SESSION_DRIVER=database
QUEUE_CONNECTION=database
CACHE_STORE=database
```

Use Stripe test keys until you are ready to go live.

## 2. Laravel Cloud (recommended)

1. Push this repo to GitHub.
2. Sign in at [cloud.laravel.com](https://cloud.laravel.com) and create a new application.
3. Connect the GitHub repository.
4. Set environment variables (Stripe keys, `APP_KEY` generated by Cloud).
5. Use **MySQL** or **PostgreSQL** in production (not SQLite).
6. Deploy — Cloud runs `composer install`, `npm run build`, and migrations.

Add a deploy hook or run once after first deploy:

```bash
php artisan migrate --force
php artisan db:seed --force
```

## 3. Railway

1. Install the [Railway CLI](https://docs.railway.app/develop/cli) or use the dashboard.
2. Create a new project from this repo.
3. Add a **PostgreSQL** plugin.
4. Set environment variables from `.env.example`.
5. Railway uses the included `Dockerfile` — build and deploy automatically.

```bash
railway login
railway init
railway add --plugin postgresql
railway up
```

Run migrations on Railway:

```bash
railway run php artisan migrate --force
railway run php artisan db:seed --force
```

## 4. Render

1. Create a **Web Service** from this repo.
2. Environment: Docker (uses root `Dockerfile`).
3. Add a **PostgreSQL** database and link `DATABASE_URL`.
4. Set Stripe and app env vars in the Render dashboard.
5. Add a **post-deploy script** or run manually:

```bash
php artisan migrate --force && php artisan db:seed --force
```

## 5. Post-deploy checklist

- [ ] `APP_URL` matches your live domain (required for Stripe redirects)
- [ ] Stripe webhook URL is registered and returning 200
- [ ] Database migrated and seeded
- [ ] `php artisan config:cache` and `route:cache` (production)
- [ ] HTTPS enabled (Stripe requires HTTPS in production)

## Local testing with Stripe

Use the [Stripe CLI](https://stripe.com/docs/stripe-cli) to forward webhooks:

```bash
stripe listen --forward-to http://example-app.test/stripe/webhook
```

Copy the webhook secret from the CLI output into `STRIPE_WEBHOOK_SECRET`.

Without Stripe keys configured, checkout uses **mock mode** (instant unlock for local development).
